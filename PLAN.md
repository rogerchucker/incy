Incy MVP Implementation Plan

 Context

 Building a greenfield incident management system (PagerDuty-inspired) from an empty repo. The
 key deliverable is a working MVP with FastAPI backend, Next.js frontend, notification worker,
 and passing Playwright E2E tests with API mocking.

 Key Decisions

 - Auth: Hardcoded seed user + X-User-Id header (simplest for MVP, no auth infra needed)
 - On-call model: service.primary_oncall_user_id + optional secondary_oncall_user_id (avoids
 separate escalation tables)
 - Data fetching: TanStack React Query
 - Notifications: DB-backed queue, SMTP to Mailpit (local dev)
 - E2E strategy: Playwright page.route() mocking — no running backend needed

 Implementation Phases

 Phase 1: Infrastructure & Scaffolding

 - /infra/docker-compose.yml (Postgres + Mailpit)
 - /Makefile with dev/test/migrate targets
 - /apps/api/ — FastAPI app skeleton, config, database, Alembic setup
 - /apps/worker/ — Python worker skeleton
 - /.gitignore

 Phase 2: Database Models & Migration

 - SQLAlchemy models: User, Team, Membership, Service, Integration, Event, Alert, Incident,
 NotificationAttempt, AuditLog
 - Alembic migration (autogenerated)
 - Seed script (team, 2 users, service, integration, sample incidents)

 Phase 3: API Endpoints

 - POST /v1/events — webhook ingestion with idempotency + dedupe
 - GET/POST /v1/services, GET/PUT /v1/services/{id} — service CRUD
 - POST /v1/services/{id}/integrations, GET /v1/services/{id}/integrations
 - GET /v1/incidents, GET /v1/incidents/{id} — list + detail
 - POST /v1/incidents/{id}/acknowledge, POST /v1/incidents/{id}/resolve
 - GET /v1/incidents/{id}/timeline — audit log
 - GET /v1/users — user list
 - Business logic: event_processor.py (dedupe→alert→incident), incident_manager.py (ack/resolve
  with row locking)
 - Middleware: error handler, rate limiter, CORS

 Phase 4: Worker

 - Poller: claims due notifications atomically, sends via SMTP
 - Retry with exponential backoff + jitter, dead-letter after max attempts
 - Escalation checker: L1→L2 after timeout

 Phase 5: Next.js Frontend

 - Scaffold with App Router, Tailwind, shadcn/ui
 - Pages: incidents list (filterable), incident detail (timeline + ack/resolve), services
 (CRUD), integrations
 - API client with TanStack React Query

 Phase 6: Playwright E2E Tests (must pass)

 - Mock all API calls via page.route()
 - incidents.spec.ts: list display, status filter, navigate to detail, show timeline, ack
 incident, resolve incident
 - services.spec.ts: list display, create service, show integrations, create integration with
 webhook example
